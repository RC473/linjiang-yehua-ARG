# 临江怪谈档案 · LINJIANG ARCHIVE

> 「有些字，只有认真找的人才看得见。」

一份意外重新上线的怪谈博客备份。原作者「林夜」三年前深夜更新后账号沉寂，此后再无音讯。这个仓库，是有人替她保留下来的最后一份镜像。

这是一个可以直接部署在 **GitHub Pages** 上的中文 ARG（现实穿越游戏）。全程 15 关以上，融合古典密码（凯撒、Atbash、摩斯电码、Base64、十六进制、二进制、维吉尼亚、栅栏密码）、图片隐写术、网页源代码线索、`robots.txt` 隐藏路径，以及 **仓库自身的 Git 提交记录**。正常游玩节奏建议以「天」为单位推进，全部解开大约需要数周的零碎时间。

## 怎么玩

1. 打开已部署好的网站（见下方部署说明），从「档案 0000」开始阅读。
2. 大部分谜题会给出一段被加密或编码过的文字，解出后在页面底部的输入框里填答案（统一使用汉语拼音全拼，不留空格）。
3. **有几关的线索不在网页正文里** —— 可能藏在图片像素里、网页源代码的注释里、CSS 隐藏的文字里，甚至藏在这个仓库自己的提交历史里。这是故意的：这个仓库本身，就是游戏的一部分。别只盯着一个页面看。
4. 每一关都有一个可展开的「提示」，卡住了可以点开看看，不会公开答案，只会给一点方向。
5. 全部解开后，最后一关需要你把之前所有关卡答案的拼音首字母按顺序拼在一起，作为最终密钥。

## 部署到 GitHub Pages

1. 新建一个 GitHub 仓库（建议设为 Public，否则大部分线索——比如提交记录——玩家看不到）。
2. 把这个项目文件夹的内容推送上去：
   ```bash
   git remote add origin https://github.com/<你的用户名>/<仓库名>.git
   git branch -M main
   git push -u origin main
   ```
   注意：本项目本地已经是一个初始化好的 Git 仓库，**提交历史本身就是第 13 关谜题的一部分**，直接 push 即可保留这些提交记录，不要 squash / 重建历史。
3. 打开仓库 Settings → Pages，Source 选择 `Deploy from a branch`，Branch 选择 `main`，文件夹选择 `/docs`，保存。
4. 稍等片刻，GitHub 会给出一个形如 `https://<你的用户名>.github.io/<仓库名>/` 的访问地址，这就是玩家看到的入口。

## 目录结构

```
.
├── docs/               ← 网站正式内容（GitHub Pages 的发布目录）
│   ├── index.html      ← 主入口（档案目录 + 谜题引擎）
│   ├── style.css
│   ├── data.js          ← 全部关卡文案与谜题数据
│   ├── app.js           ← 交互引擎（答案校验、进度保存）
│   ├── robots.txt
│   ├── 404.html
│   ├── assets/          ← 图片等素材
│   └── log/              ← 独立的「原始博客文章」页面
├── tools/                ← 辅助工具（不在网站上被链接，需要自己发现）
└── README.md
```

## 给部署者的提醒

- 游戏答案的校验完全在浏览器本地完成（SHA-256 哈希比对），源码里不会出现任何明文答案，玩家没法直接看源码作弊。
- 进度保存在玩家浏览器的 `localStorage` 里，换设备或清空浏览器数据会重置进度，这是纯静态网站的天然限制，可以在发布公告里提前说明。
- 如果你想在自己的社群里发布这个游戏，可以把仓库地址和一句引导语（比如「有人在我的关注列表里发现了这个」）发出去，让第一步的发现本身也成为体验的一部分。
- 想要修改剧情、增删关卡，直接编辑 `docs/data.js` 即可，每一关的字段说明可以参考文件内已有的结构。
